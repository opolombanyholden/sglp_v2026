Architecture et accès
URLs de connexion, rôles utilisateurs, structure du tableau de bord administrateur.

1.1 URLs de connexion

Le SGLP dispose de deux interfaces distinctes selon le profil utilisateur :

🔐 Espace Administrateur / Agent

Accès à la gestion complète des dossiers, référentiels, CMS, API et rapports.

/admin/login
admin agent

👤 Espace Opérateur

Accès à la soumission et au suivi des dossiers d'une organisation.

/login
operator
⚠️ Important Les deux URLs de connexion sont distinctes et volontaires. Un compte admin ne peut pas se connecter via /login et inversement.
🖼️
Capture d'écran
Page de connexion administrateur
/admin/login

2.3 Génération du récépissé

À l'approbation, le système génère automatiquement :

  • Récépissé PDF officiel avec numéro unique, QR code d'authentification, données de l'organisation et membres du bureau
  • QR code actif dans la table qr_codes pointant vers l'organisation
  • L'organisation devient visible dans l'annuaire public
ℹ️ Format du numéro de récépissé TYPE-ANNÉE-NUMÉRO — exemple : ASS-2026-000042 pour une association. Préfixes : ASS association · ONG ONG · PP parti politique · CR confession religieuse.
🖼️
Capture d'écran
Aperçu du récépissé PDF généré avec QR code
/admin/dossiers/{id}/recepisse

4.3 Vérification d'un récépissé

URL : /annuaire/verify/{code} — accessible via scan QR, saisie manuelle ou lien direct.
Throttle : 20 requêtes/minute par IP pour prévenir le scraping.

Trois types de code acceptés :

  • Code QR (scanné depuis un document officiel)
  • Numéro de récépissé (ex : ASS-2026-000042)
  • ID numérique de l'organisation

✅ Récépissé VALIDE

Organisation trouvée, statut actif. Bannière verte avec fiche complète.

⚠️ Organisation SUSPENDUE

Récépissé authentique mais organisation suspendue. Bannière orange.

❌ NON RECONNU

Code introuvable. Avertissement fraude potentielle, bouton de signalement.

🚫 RÉCÉPISSÉ INVALIDE

Organisation radiée ou rejetée. Récépissé existant mais plus en vigueur.

🖼️
Capture d'écran
Page de résultat de vérification — récépissé valide (bannière verte)
/annuaire/verify/ASS-2026-000001

API Interopérabilité V1
Permettre aux systèmes d'information tiers d'accéder aux données officielles de l'annuaire SGLP de façon sécurisée et tracée.

5.1 Gestion des tokens API

Accès : Administration → API / Interopérabilité (icône 🔌 dans la barre latérale)

🖼️
Capture d'écran
Liste des tokens API — statut, utilisation, dernière connexion
/admin/api/tokens

Créer un nouveau token

1
Cliquer sur "Nouveau token" Renseigner le nom du système client et l'organisation demandeuse.
2
Sélectionner les permissions (scopes) organisations verify stats * (accès complet)
3
Définir le rate limit De 10 à 600 requêtes par minute. Recommandé : 60 req/min.
4
Copier immédiatement le token généré Le token brut n'est affiché qu'une seule fois. Seul le hash SHA-256 est stocké en base.
🔴 Token perdu = token révoqué Si le token brut est perdu, il faut en créer un nouveau. Il est impossible de récupérer la valeur originale.
🖼️
Capture d'écran
Formulaire de création d'un token + affichage unique du token généré
/admin/api/tokens/create

5.2 Endpoints disponibles

Base URL : https://www.sglp.ga/api/v1/public

MéthodeEndpointScopeDescription
GET /organisations organisations Liste paginée avec filtres
GET /organisations/{id} organisations Détail + membres bureau
GET /organisations/verify/{code} verify Vérification récépissé / QR

Exemple de requête

GET /api/v1/public/organisations?type=ong&province=Estuaire
Authorization: Bearer votre_token_ici
Accept: application/json

5.4 Révoquer un token compromis

1
Aller dans Tokens API : /admin/api/tokens
2
Cliquer sur "Révoquer" en face du token concerné. Le token est immédiatement désactivé.
3
Créer un nouveau token et le transmettre au système client par canal sécurisé.

# Régénérer les QR codes manquants ou corrompus

php artisan qr:regenerate --type=organisation

8.1 Tableau de diagnostic rapide

SymptômeCause probableAction
Page blanche ou erreur 500 Erreur PHP / cache corrompu php artisan cache:clear && view:clear
Images portail absentes Lien symbolique storage manquant php artisan storage:link
QR codes non lisibles Fichiers PNG manquants php artisan qr:fix-missing-png
PDF ne génère pas Permissions storage incorrectes chmod -R 775 storage/
API retourne 401 Token révoqué ou expiré Créer un nouveau token dans l'admin
API retourne 429 Rate limit dépassé Augmenter le rate_limit du token concerné
Connexion admin impossible Session expirée ou CSRF Vider le cache navigateur, réessayer
Emails non reçus Configuration SMTP Vérifier MAIL_HOST et MAIL_PASSWORD dans .env

8.2 Commandes de maintenance essentielles

# ── Vider tous les caches ──
php artisan cache:clear
php artisan config:clear
php artisan route:clear
php artisan view:clear

# ── Reconstruire les caches (production) ──
php artisan config:cache
php artisan route:cache
php artisan view:cache

# ── Mode maintenance ──
php artisan down --message="Maintenance en cours."
php artisan up

# ── Vérifier l'état de l'application ──
php artisan qr:health-check
tail -50 storage/logs/laravel.log | grep -E "ERROR|CRITICAL"
